# Системные требования QMS-DSM

## 📋 Обзор

QMS-DSM — это веб-приложение для контроля качества и управления процессом инспекции на производстве. Приложение состоит из:
- **Frontend**: React 18 + Vite (раздается через Nginx)
- **Backend**: Node.js + Express + SQLite
- **Infrastructure**: Docker (для Nginx), Linux systemd

---

## 🔧 Минимальные системные характеристики

### Для сервера/хоста (минимум)

| Компонент | Требование |
|-----------|-----------|
| **ОС** | Linux (Ubuntu 20.04 LTS, CentOS 8 или выше; с поддержкой systemd) |
| **Процессор** | 2 ядра × 2.0 GHz (Intel Xeon, AMD EPYC или аналог) |
| **ОЗУ** | 2 GB |
| **Дисковое пространство** | 5 GB (для приложения, БД SQLite и логов) |
| **Сетевой интерфейс** | 100 Mbps Ethernet |

### ПО (обязательно)

| Компонент | Минимальная версия |
|-----------|---|
| **Node.js** | 18.x LTS |
| **npm** | 9.x или выше |
| **Docker** | 20.10+ |
| **Docker Compose** | 1.29+ (опционально) |

### Сетевые требования (минимум)

- Открытые порты: **80 (HTTP)**, **3001 (Backend API)**
- Доступ в интернет: для загрузки npm-пакетов при установке
- Локальное подключение: Backend и Nginx должны находиться на одном хосте

## ⭐ Рекомендуемые системные характеристики

### Для продакшена (рекомендуется)

| Компонент | Рекомендация |
|-----------|-----------|
| **ОС** | Linux (Ubuntu 22.04 LTS, AlmaLinux 9 или выше) |
| **Процессор** | 4+ ядра × 2.5 GHz (современный Intel Xeon или AMD EPYC) |
| **ОЗУ** | 8 GB или больше |
| **Дисковое пространство** | 20+ GB (SSD для лучшей производительности) |
| **Сетевой интерфейс** | 1 Gbps Ethernet (гарантированное подключение) |

### ПО (рекомендуется)

| Компонент | Рекомендованная версия |
|-----------|---|
| **Node.js** | 20.x LTS или выше |
| **npm** | 10.x или выше |
| **Docker** | 24.0+ |
| **Docker Compose** | 2.x |

### Дополнительно для продакшена

- **Reverse Proxy**: Nginx/Apache перед фронтенд слоем
- **SSL/TLS**: Сертификат SSL для HTTPS
- **Мониторинг**: Prometheus, Grafana или аналог
- **Логирование**: ELK Stack или централизованное хранилище логов
- **Backup**: Автоматическое резервное копирование БД (минимум ежедневно)

## 💻 Требования для клиентских устройств

### Для веб-браузера (инспекторов, операторов)

| Параметр | Минимум | Рекомендуется |
|----------|---------|---|
| **Браузер** | Chrome 90+, Firefox 88+, Safari 14+ | Chrome/Firefox последняя версия |
| **ОЗУ браузера** | 512 MB | 2+ GB |
| **Процессор** | Одноядерный 1.5 GHz | Двухядерный 2+ GHz |
| **Подключение** | 5 Mbps (для базовой работы) | 10+ Mbps (для real-time функций) |
| **Разрешение экрана** | 1024×768 px | 1920×1080 px (или планшет 10"+) |

### Мобильные устройства (планшеты для контроля)

- **ОС**: iOS 14+ или Android 10+
- **ОЗУ**: 2+ GB
- **Экран**: 7–10 дюймов (минимум 1024×768)
- **Подключение**: Wi-Fi 5 GHz или 4G LTE

## 🗄️ Требования к БД (SQLite)

| Параметр | Значение |
|----------|---------|
| **СУБД** | SQLite 3.32+ |
| **Размер БД** | 50 MB – 500 MB (в зависимости от объема логов) |
| **Место на диске** | +20% от размера БД для WAL и временных файлов |
| **Режим подключения** | Write-Ahead Logging (WAL) |

### Сценарии масштабирования

- **До 100 пользователей**: Текущей конфигурации достаточно
- **100–500 пользователей**: Рекомендуется увеличить ОЗУ до 16 GB и перейти на SSD
- **500+ пользователей**: Рассмотреть миграцию на PostgreSQL

## 📊 Производительность и нагрузки

### Поддерживаемые одновременные пользователи

| Сценарий | Конфигурация | Пиковая нагрузка |
|----------|-----------|---|
| **Разработка** | 2-ядерный CPU, 2 GB RAM | ~5–10 пользователей |
| **Малое производство** | 4-ядерный CPU, 8 GB RAM | 20–50 пользователей |
| **Среднее производство** | 8-ядерный CPU, 16 GB RAM | 100–200 пользователей |
| **Крупное производство** | 16+ ядер, 32+ GB RAM | 500+ пользователей |

### Требования к пропускной способности API

- **Базовая операция**: ~100 KB/запрос
- **Real-time SSE (Heartbeat)**: ~1 KB/15 сек на пользователя
- **Загрузка фото дефектов**: ~500 KB – 2 MB на фото (с сжатием на клиенте)

## 🚀 Установка и развертывание

### Минимальные шаги развертывания

```bash
# 1. Установка зависимостей ОС
sudo apt-get update
sudo apt-get install -y nodejs npm docker.io

# 2. Клонирование репозитория
git clone https://github.com/nrkfo/qms-dsm.git
cd qms-dsm

# 3. Запуск установочного скрипта
chmod +x install.sh
./install.sh
```

Подробнее: см. [README.md](./README.md)

## 🔒 Рекомендации безопасности

- ✅ Использовать HTTPS в продакшене
- ✅ Регулярно обновлять Node.js, npm и Docker
- ✅ Ограничить доступ к портам (использовать firewall)
- ✅ Включить аутентификацию 2FA для администраторов
- ✅ Регулярно создавать резервные копии SQLite БД
- ✅ Мониторить логи backend.log на предмет ошибок
