# QMS (Quality Management System)

Система для контроля качества и управления процессом инспекции на производстве. QMS автоматизирует процессы технического контроля: от входного контроля поступающих материалов и комплектующих (IQC) до финального контроля готовой продукции и этикеток (OQA).

## 🛠 Последнее обновление

- **Мгновенное обновление системы**: Добавлена возможность автоматического скачивания обновлений с GitHub и перезапуска сервера прямо из панели администратора в 1 клик.
- **Интеграция с Google Drive**: Автоматическое резервное копирование базы данных по расписанию прямиком в облако.
- **Коммерческая лицензия и Ребрендинг**: Платформа переименована в QMS. Добавлена проприетарная лицензия, защищающая авторские права и запрещающая несанкционированное копирование или аренду.
- **Оптимизация производительности**: Устранены мерцания графиков, отключены тяжелые анимации, оптимизировано логирование и снижена нагрузка на CPU.
- **Мониторинг активности**: Внедрено отслеживание статуса пользователей (онлайн/оффлайн) в реальном времени.

## 💻 На чем это работает (Tech Stack)

Система построена на современных веб-технологиях:
- **Фронтенд**: React, TypeScript, Vite, Zustand.
- **Бэкенд**: Node.js, Express, TypeScript.
- **База Данных**: SQLite 3 (локальная реляционная СУБД с поддержкой WAL и виртуальных колонок).
- **Инфраструктура**: Linux (systemd), Docker (для Nginx).

## 📊 База Данных и Структура

Система базируется на СУБД SQLite 3:

- `users` — учетные записи пользователей, роли (Admin, Inspector и др.) и права доступа.
- `lots` — производственные партии и заказы.
- `tv_models`, `suppliers`, `articles` — справочники моделей, поставщиков и номенклатуры комплектующих.
- **Таблицы логов** (`oqa_tv_logs`, `iqc_aql_logs` и т.д.) — хранят результаты инспекций в формате динамической JSON-структуры с использованием высокоскоростных B-Tree индексов для мгновенной генерации KPI и дашбордов.
- `audit_logs` — журнал действий (кто, что и когда изменил) для обеспечения полной прослеживаемости.

## ⚙️ Рекомендуемые системные требования

- **ОС**: Linux (Ubuntu 20.04+ или Debian 11+).
- **Процессор**: 2 ядра (x86_64 или ARM64).
- **Оперативная память**: 4 ГБ ОЗУ или больше.
- **Накопитель**: 20 ГБ SSD.
- **ПО**: Установленные Node.js (v18+), npm, Git и Docker.

## 🚀 Как запустить

Для установки зависимостей, сборки и автозапуска системы, просто перейдите в папку с проектом и запустите скрипт:

```bash
chmod +x install.sh
./install.sh
```

**Что делает скрипт установки:**
1. Устанавливает Node-пакеты для Backend.
2. Собирает production-бандл Frontend.
3. Копирует файл сервиса и добавляет бэкенд в автозапуск как фоновую службу Linux (`qms-backend`).
4. Запускает Nginx внутри Docker для проксирования API и раздачи фронтенда на 80 порту.

### Полезные команды для обслуживания:
- Проверить статус бэкенда: `sudo systemctl status qms-backend`
- Ручной перезапуск бэкенда: `sudo systemctl restart qms-backend`
- Логи бэкенда: `tail -f backend.log`
- Перезапустить Nginx (фронтенд): `sudo docker restart qms-nginx`
